Что такое уведомления?

Уведомления в Docebo LMS - это автоматические сообщения, которые отправляются пользователям при определенных событиях. Например:

  • Когда пользователь записывается на курс
  • Завершает обучение
  • Приближается дедлайн
  • Начинается вебинар или ILT сессия

🚀 Базовые компоненты уведомления

  • Событие - что запускает уведомление
  • Получатели - кто получает сообщение
  • Канал доставки - email, в платформе или push
  • Сообщение - текст уведомления
  • Время - когда отправлять

Просмотр списка уведомлений

Путь: Admin menu > Notifications > Manage

Для начала работы перейдите в Admin menu > Notifications > Manage. Здесь вы увидите список всех уведомлений на платформе.

Список уведомлений показывает тип события, каналы доставки и статус

Что вы видите в списке:

  • Event type - тип события, которое запускает уведомление
  • Иконки каналов доставки - email, платформа, push
  • Status - активно ли уведомление (Active/Inactive)
  • Иконка Play - для ручного запуска некоторых уведомлений
Полезно знать: Только активные уведомления отправляются пользователям. Используйте фильтры и поиск для быстрого поиска нужного уведомления.

Настройка уведомления

Чтобы настроить уведомление, нажмите на его название или создайте новое кнопкой +. Откроется страница конфигурации с тремя основными вкладками.

Интерфейс настройки с вкладками Properties, Recipients и Trigger filters

Основные шаги настройки:

  1. 1Recipients - выберите, кто получит уведомление
  2. 2Properties > Composition - создайте сообщение
  3. 3Properties > Details and timing - настройте время отправки
  4. 4Trigger filters - ограничьте область действия (опционально)

Типы получателей:

  • Users - пользователи, которых касается событие
  • Managers - менеджеры этих пользователей
  • Instructors - преподаватели курса
  • Superadmins - все администраторы

Создание сообщения

На вкладке Properties > Composition вы создаете содержание уведомления для разных каналов доставки.

Настройка email уведомлений и платформенных уведомлений

Настройка Email уведомления:

  1. Включите "Enable email as delivery channel"
  2. Укажите имя и email отправителя
  3. Введите тему сообщения
  4. Напишите текст сообщения

Использование динамических переменных:

В сообщениях можно использовать shortcodes, которые автоматически заменяются данными:

  • {{first_name}} - имя пользователя
  • {{course_name}} - название курса
  • {{course_link}} - ссылка на курс
  • {{due_date}} - дата дедлайна
Пример email:
Тема: Добро пожаловать на курс {{course_name}}

Сообщение: Здравствуйте, {{first_name}}! Вы записаны на курс "{{course_name}}". Начать обучение: {{course_link}}

Планирование времени отправки

На вкладке Properties > Details and timing настройте, когда отправлять уведомление.

Интерфейс настройки времени отправки уведомлений

Основные варианты времени:

  • At the time of the event - сразу при наступлении события
  • Before the event - за определенное время до события
  • After the event - через определенное время после события
  • Manually - отправка вручную

Примеры использования:

  • Сразу при записи: "At the time of the event" для приветственных сообщений
  • Напоминание о курсе: "1 day before" для напоминания о начале ILT
  • Благодарность за завершение: "At the time of the event" при завершении курса
Рекомендация: Для большинства уведомлений используйте "At the time of the event" - это самый простой и надежный вариант.

Продвинутые возможности уведомлений

Эта секция описывает дополнительные функции для опытных пользователей.

Фильтры триггеров

Позволяют ограничить уведомления определенными группами пользователей или курсами. Используйте вкладку Trigger filters для настройки ограничений по:

  • Groups - определенные группы пользователей
  • Branches - определенные филиалы организации
  • Courses - определенные курсы или планы обучения
Интерфейс настройки фильтров триггеров для ограничения области действия уведомлений

Digest уведомления (сводные)

Группируют несколько событий в одно сообщение вместо отправки множества отдельных уведомлений:

  • Еженедельные сводки - все записи на курсы за неделю
  • Ежемесячные отчеты - завершенные курсы за месяц
  • Уведомления о дедлайнах - все приближающиеся сроки
Настройки digest уведомлений для группировки событий в сводные сообщения

Community уведомления

Автоматически создаваемые уведомления для сообществ:

  1. Перейдите в Admin menu > COMMUNITIES > Communities hub
  2. Выберите сообщество
  3. На вкладке Settings найдите раздел Notifications
  4. Нажмите "Create community notifications"
Интерфейс управления сообществами и настройка уведомлений для сообществ

Мобильные push-уведомления

Настройка уведомлений для мобильного приложения:

  • Доступны только для определенных событий
  • Ограничения: 50 символов для заголовка, 150 для текста
  • Поддержка shortcodes в тексте сообщения

Проверка логов уведомлений

Отслеживание отправленных уведомлений через вкладку Logs:

  • Просмотр всех отправленных уведомлений
  • Фильтрация по датам (максимум 14 дней за раз)
  • Статус доставки и время отправки
Интерфейс просмотра логов отправленных уведомлений с фильтрацией и статусами доставки

Многоязычные уведомления

Создание уведомлений на нескольких языках:

  1. Включите опцию "Translate fields"
  2. Выберите язык из выпадающего списка
  3. Введите перевод темы и текста сообщения
  4. Повторите для всех необходимых языков

Советы по оптимизации

Производительность:
  • Деактивируйте уведомления перед массовыми операциями
  • Используйте digest для групповых уведомлений
  • Тестируйте уведомления на небольшой группе сначала
Ограничения и особенности:
  • Максимум 65535 символов на уведомление
  • Email адреса должны быть настроены в Domain management
  • Планировщик проверяет события в начале каждого часа
  • Power Users видят только уведомления для назначенных ресурсов

Лучшие практики

  • Тестирование: Создайте тестовую группу пользователей
  • Именование: Используйте понятные имена уведомлений
  • Персонализация: Активно используйте shortcodes
  • Редактирование: Деактивируйте уведомление перед изменениями
  • Мониторинг: Регулярно проверяйте логи отправки

What are Notifications?

Notifications in Docebo LMS are automatic messages sent to users when specific events occur. For example:

  • When a user enrolls in a course
  • Completes training
  • A deadline approaches
  • A webinar or ILT session starts

🚀 Basic Notification Components

  • Event - what triggers the notification
  • Recipients - who receives the message
  • Delivery channel - email, in-platform, or push
  • Message - notification text
  • Timing - when to send
